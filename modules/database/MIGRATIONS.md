# Database Migration Guide

This guide explains how to set up and run database migrations using Drizzle ORM with Cloudflare D1.

Migration files are generated by **drizzle-kit** and applied by **wrangler** — no custom scripts needed.

## Quick Start

### 1. Initialize Database Dependencies

```bash
# From root workspace
pnpm install
```

### 2. Create D1 Database in Cloudflare

```bash
cd modules/backend

pnpm wrangler d1 create demo-db
```

Output will show:
```
✅ Successfully created D1 database demo-db
database_id = "12345678-1234-1234-1234-123456789012"
```

### 3. Update wrangler.jsonc

Edit `modules/backend/wrangler.jsonc` and set the real `database_id`:

```jsonc
{
  "d1_databases": [
    {
      "binding": "UPLOADS_DB",
      "database_name": "demo-db",
      "database_id": "12345678-1234-1234-1234-123456789012",
      "migrations_dir": "../database/drizzle"
    }
  ]
}
```

### 4. Generate Migrations from Schema

```bash
cd modules/database

pnpm run migrate:generate
```

This creates SQL files in `modules/database/drizzle/`.

### 5. Apply Migrations

**Local dev (wrangler local D1):**

```bash
pnpm run db:apply:local
```

**Local dev (SQLite file, faster for unit tests):**

```bash
cd modules/database
pnpm run migrate:local
```

**Remote Cloudflare D1:**

```bash
pnpm run db:apply
```

Wrangler tracks applied migrations automatically in the `d1_migrations` table and skips already-applied files.

List migration status at any time:

```bash
cd modules/backend
pnpm wrangler d1 migrations list demo-db          # remote
pnpm wrangler d1 migrations list demo-db --local  # local
```

## File Structure

```
modules/database/
├── src/
│   ├── users/schema.ts
│   ├── cases/schema.ts
│   └── auth-accounts/schema.ts
├── drizzle/                      # Auto-generated migration files
│   ├── 0000_*.sql
│   ├── 0001_*.sql
│   └── meta/                     # Drizzle Kit metadata
└── drizzle.config.ts            # Drizzle Kit config
```

## Modifying the Schema

### 1. Edit the TypeScript Schema

Edit the relevant file in `modules/database/src/`.

### 2. Generate New Migration

```bash
cd modules/database
pnpm run migrate:generate
```

### 3. Apply Locally

```bash
pnpm run db:apply:local
# or for SQLite dev DB:
pnpm run migrate:local
```

### 4. Deploy to D1

```bash
pnpm run db:apply
```

## Common Scenarios

### Adding a New Table

1. Add schema file in `modules/database/src/`
2. Register it in `drizzle.config.ts` schema array
3. `pnpm run migrate:generate` → `pnpm run db:apply:local`

### Modifying a Column

1. Edit the schema
2. `pnpm run migrate:generate` → `pnpm run db:apply:local`

### Removing a Column

SQLite has limited `ALTER TABLE` support — drizzle-kit will generate a table-recreate migration automatically.

### Adding an Index

Add the index in the schema definition, then generate and apply.
